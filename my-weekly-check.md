# my-weekly-check — C-제조 주간 점검 루틴

**담당**: 송민우 (C-제조) | **주기**: 매주 월요일 오전

> 이 파일의 각 항목 프롬프트를 Claude Code에 그대로 붙여넣어 실행합니다.

---

## 1. 지난주 불량 패턴 변화 확인

```
@practice-data/C-제조/defect_log.csv 에서
지난 5일(최근 날짜 기준) 라인별·유형별 불량 건수를 표로 정리하고
전주 대비 증감이 있는 항목을 강조해줘. CLAUDE.md 5분류 기준 적용.
```

---

## 2. 생산량 달성률 점검

```
@practice-data/C-제조/production.csv 에서
이번 주 라인별 생산목표 대비 실생산량 달성률(%)을 계산하고
목표 미달 라인과 원인 가설을 간단히 정리해줘.
```

---

## 3. 불량 TOP3 집중 분석

```
@practice-data/C-제조/defect_log.csv 에서
이번 주 불량 건수 상위 3개 (라인+유형 조합)를 뽑아
각 항목별 발생 시간대 패턴과 권장 조치를 한 줄씩 제시해줘.
```

---

## 4. 금형 교체 일정 확인

```
@practice-data/C-제조/mold_history.csv 를 읽고
이번 주 교체 예정 금형이 있는지 확인하고
지난 교체 후 불량률 변화 추이를 간단히 요약해줘.
```

---

## 5. 팀장 보고용 주간 요약 초안

```
@defect_analysis.md 와 @practice-data/C-제조/production.csv 를 참고해서
이번 주 팀장 보고용 주간 품질 요약을 3개 항목으로 작성해줘:
1. 핵심 지표 (불량률, 생산 달성률)
2. 이번 주 주요 이슈
3. 다음 주 조치 계획
형식은 표로 정리해줘.
```

---

## 6. 개선 아이디어 정리

```
지난 주 @manufacturing_review.md 의 개선 제안 중
이번 주에 실행 가능한 항목 1~2개를 골라
구체적인 실행 방법(담당자, 기한, 확인 방법)을 표로 정리해줘.
```

---

## 7. 주간 루틴 마무리 저장

```
오늘 생성한 분석 결과를 weekly_report_YYYYMMDD.md 파일로 저장하고
저장해줘 → 올려줘 순서로 GitHub에 업로드해줘.
```

---

> **사용 팁**: 매주 실행하면서 프롬프트 문장이 어색하거나 결과가 부족하면 그 항목만 한 줄 수정합니다.
> 한 번에 완벽할 필요 없습니다. 매주 조금씩 다듬어가는 것이 하네스의 힘입니다.

*하네스 엔지니어링 = 실수가 불가능한 환경 설계 = 공장 안전 인터록*
