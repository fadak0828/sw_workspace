# A/B/C 비교 실험 결과 — C-제조 (송민우)

## 실험 과제: defect_log.csv 불량 분석

| 방식 | 입력 | 결과 특징 | 재사용성 |
|------|------|-----------|---------|
| **A — 지시만** | "분석해줘" | 뭘 분석할지 모르는 답변, 기준 없음 | ❌ 낮음 |
| **B — 지시 + CLAUDE.md** | "@defect_log.csv 분석해줘" | 5분류 기준 자동 반영, 표 형식 출력 | ✅ 중간 |
| **C — 지시 + CLAUDE.md + Plan** | Plan 모드 + "@defect_log.csv 라인별 불량률 집계해서 품질보고서.md 만들어줘" | 계획 확인 후 체계적 실행, 재현 가능 | ✅✅ 높음 |

> **핵심**: A → B → C는 모델이 아닌 **구조**의 차이

## 프롬프트 4법칙 적용 예시

| 법칙 | 나쁜 예 | 개선 예 |
|------|--------|--------|
| 구체적으로 | "불량 정리해줘" | "@defect_log.csv 에서 LINE-1 치수불량만 추출해줘" |
| 한 번에 하나씩 | "분석하고 보고서 만들고 팀장에게 보내줘" | 단계별 순차 요청 |
| 결과 형태 명시 | "설명해줘" | "표로 정리해서 defect_summary.md로 저장해줘" |
| @ 파일 지정 | "CSV 분석해줘" | "@practice-data/C-제조/defect_log.csv 분석해줘" |
